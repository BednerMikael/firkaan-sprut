Закупка товаров

Поставщики и параметры, каталог цен, заказы на поставку, контроль прохождения, взаиморасчёты, заявки, консигнация.

Менеджер по закупкамОператорПоставщики
Введение

Управление закупками в ПС «СПРУТ» производится на уровне Заказов на поставку. Заказ — список товаров с количествами и ценами от конкретного поставщика. Взаиморасчёты с поставщиками контролируются по заказам (не по накладным). Работа ведётся в модуле «Поставки».

Раздел 1

Поставщики и их параметры

ПараметрЗначения / Описание
Вид поставщикаИмпортный (или Неплательщик НДС) / Отечественный / Внутренний (компания группы)
Вариант оплатыПредоплата полная / Предоплата частичная / С отсрочкой платежа / По реализации (консигнация)
Макс. задолженностьЛимит долга перед поставщиком. При превышении — регистрация новой оплаты блокируется.
Макс. отсрочка (дней)Срок погашения долга при отсрочке. Для консигнации — периодичность платежей.
Валюта расчётовВ какой валюте ведутся взаиморасчёты.
% предоплатыДля варианта «Частичная предоплата».
ℹ️
Ввод и редактирование поставщиков — в Справочнике «Клиентов и поставщиков» (Глава 13. Справочники).
Раздел 2

Каталог цен поставщиков

Журнал «Каталог цен поставщиков» — для хранения и анализа цен по каждому товару от всех поставщиков.

Цены обновляются автоматически при каждом оприходовании от поставщика.
Ввод вручную — для хранения предложений без фактического прихода.
Сравнение цен разных поставщиков на один товар — для переговоров.
Цены из каталога — первоначальные при создании нового заказа.

Модуль «Прайс-листы поставщиков и конкурентов» — для импорта прайс-листов из Excel и сравнения предложений поставщиков с текущими ценами каталога.

Раздел 3

Журнал заказов на поставку

Создание заказа

1
Модуль «Поставки» → Журнал «Заказов на поставку» → «Добавить».
2
Укажите: Поставщик, Торговая площадка, Вид заказа (Импортный / Отечественный / Внутренний).
3
Заполните даты: подтверждения, начала отсрочки, ожидаемой доставки.
4
Добавьте товары в табличную часть: по штрих-коду, артикулу или из каталога цен поставщика.
5
Укажите количество и цену для каждой позиции → стоимость заказа рассчитается автоматически.
6
Сохраните — заказ в статусе «Готовится».

Статусы заказа на поставку

СтатусОписание
ГотовитсяЧерновик — можно редактировать и удалять.
СоставленПодтверждён — начинается контроль прохождения.
ВыполняетсяТовар в пути или на таможне.
Сформирована приходная накладнаяПо заказу создана приходная накладная.
ВыполненТовар получен, взаиморасчёты урегулированы.

Контроль прохождения по типу поставщика

Отечественный — контроль расходной / налоговой накладной поставщика. Срок поставки обычно 1–5 дней.
Импортный — дополнительно контролируется ВМД (таможенная декларация), дата растаможивания, прибытие на таможню.
Внутренний — для перемещений внутри группы компаний. Контроль передаточной накладной.

Связь заказа с приходной накладной

Автоматическое создание прихода — кнопка «Сформировать приходную накладную» в заказе → создаётся черновик накладной в модуле «Приход».
Независимый приход — накладная создаётся в «Приходе» отдельно, потом привязывается к заказу при проведении.
Авто-создание заказа из прихода — если заказ не был создан заранее, он генерируется автоматически при оприходовании накладной.
Раздел 4

Регистрация и контроль оплат

Варианты оплаты поставщику

ВариантЛогика
Предоплата 100%Оплата до поставки. Без лимита задолженности.
Предоплата частичнаяЧасть до поставки (%), остаток — по получении товара.
С отсрочкой платежаОплата в течение N дней после поставки. Система контролирует лимит задолженности.
По реализации (консигнация)Оплата по факту продажи. Периодичность — в параметрах поставщика.

Способы оплаты заказа

Прямая оплата (наличные) — указывается сумма платежа напрямую в заказе.
Оплата по платёжке — привязка безналичного платежного поручения из журнала «Исходящие платежи» модуля «Финансы».
Оплата с остатка на счету — средства накопленные на лицевом счету поставщика (от предыдущих возвратов или авансов).
⚠️
При превышении максимально разрешённой задолженности — регистрация новой оплаты с отсрочкой блокируется. Для разблокировки: либо увеличьте лимит, либо оплатите предыдущие заказы.

Журнал состояния счетов поставщиков

Модуль «Финансы» → Журнал «Состояние счетов поставщиков» — текущий баланс по каждому поставщику:

Поступления на счёт — зачисления: авансы, возвраты товаров.
Убытия со счёта — регистрации оплат с отсрочкой (долг перед поставщиком).
Оплаты с остатка — использование накопленных средств для оплаты заказов.
Автоматическая разноска остатков по заказам — кнопка «Разноска остатков на счету».
Раздел 5

Заявки на поставку

Заявка — предварительный документ до оформления заказа. Удобна для планирования и согласования.

1
Журнал «Заявки на поставку» → «Добавить» → выберите поставщика.
2
Способ формирования списка товаров:
• Вручную — добавьте товары из каталога.
• Из анализа — кнопка «Из интенсивности продаж» → система сама предложит товары с низкими остатками.
• Из Excel — импорт списка из прайс-листа поставщика.
3
Согласуйте заявку → кнопка «Создать заказ на поставку» → заявка конвертируется в заказ.
Раздел 6

Торговля по условиям комиссии (консигнация)

При варианте оплаты «По реализации» поставщик передаёт товар без предоплаты, оплата — по факту продажи.

Приходование комиссионных товаров — обычная приходная накладная, но тип оплаты «По реализации».
Акты комиссии — формируются за период на основании реализованных товаров.
Контроль задолженности — в журнале «Состояние счетов поставщиков» тип «По реализации».

Глава 10 · Справочное руководство

Система ценообразования и скидок

Дилерские категории, наценки, расценка при приходе, барьер удержания, цены при продаже, акционное ценообразование.

Категорийный менеджерОператорДиректорМенеджер по закупкам
Введение

ПС «Спрут» реализует гибкую систему ценообразования, максимально исключающую ручное вмешательство операторов в процесс формирования цен. Процесс состоит из трёх этапов: формирование ценовой политики → расценка → формирование цены при продаже.

Раздел 1

Формирование ценовой политики

Дилерские категории

Дилерские категории — уровни цен реализации (Розница, Опт, VIP, Оптовая база и т.д.). Для каждой категории задаются цены по всем товарам. Категория присваивается каждому клиенту при вводе в систему.

ℹ️
Дилерские категории имеют ценовые барьеры — при превышении суммы заказа порогового значения цены пересчитываются по более льготной категории автоматически.

Цены специального предложения

Цены спецпредложения — скидки от количества на конкретные товары. Задаётся: категория спецпредложения, объём (с какого количества действует), цена. Клиенты получают эти цены автоматически при достижении объёма покупки.

Наценки

Для каждой ценовой категории задаются проценты наценки: по группам товаров, по производителям, индивидуально по товару. Наценки хранятся в Справочнике «Процентов наценки и ценовых категорий».

Фиксированные и индикативные цены

Тип ценыОписаниеПоведение при расценке
ФиксированнаяУстанавливается производителем. Не меняется автоматически.Расценка не производится — цена остаётся фиксированной
ИндикативнаяГосударственная максимальная ценаРассчитанная цена не может превышать индикатив

Ценовые договора

С конкретным клиентом можно заключить ценовой договор — фиксированные цены на определённые товары на заданный период. Договорные цены имеют приоритет над обычным расчётом цен по дилерской категории.

Раздел 2

Расценка — автоматический расчёт цен при приходе

Принципы расценки

Базовая формула: Рассчитанная цена = Исходная цена × (1 + % наценки / 100)

Схема НДС «Вложенный» — рассчитывается цена с НДС, цена без НДС = цена с НДС / (1 + ставка НДС/100).
Схема НДС «Наложенный» — рассчитывается цена без НДС, цена с НДС = цена без НДС × (1 + ставка НДС/100).
Контроль фиксированных цен — если цена помечена как фиксированная, расценка не выполняется.
Прибавление фиксированной суммы — к рассчитанной цене можно добавить фиксированную сумму (стоимость этикетки, упаковки). Задаётся через свойство товара.

Барьер удержания цены реализации (%)

Если новая приходная цена изменилась менее чем на Х% по сравнению с предыдущей — цена реализации не пересчитывается. Предотвращает постоянные незначительные изменения цен на полке.

💡
Задаётся в карточке товара (на уровне подгруппы). Активируется в Свойствах ПС «Спрут» → «Настройки ценообразования» → «Учитывать процент удержания цен реализации».

Округление цен реализации

ПараметрОписание
Барьер округленияЦена, ниже которой действует одна кратность, выше — другая
Округление до барьераКратность для дешёвых товаров (напр., 0.10 тенге)
Округление после барьераКратность для дорогих товаров (напр., 0.25 или 1.00 тенге)
В большую сторонуДа — цена только увеличивается; Нет — по математическим правилам

Универсальный режим расценки

Вызывается из журнала Приходных накладных при проведении. Показывает для каждого товара:

Приходную цену, рассчитанную цену реализации, текущую цену, плановую наценку, отклонение от плановой.
Остатки на складе — для оценки влияния изменения цены на стоимость склада.
Цветовое выделение товаров — сигнал о проблемных ситуациях (отклонение наценки, фиксированная цена и т.д.).
Ручное редактирование цены прямо в режиме расценки — при необходимости.
Кнопка «Утвердить» — цены становятся активными.

Раздел 3

Формирование цены при продаже

Специальные опции ценообразования для клиентов

ОпцияПоведение
Не задействоватьСтандартный расчёт по дилерской категории
Цена не меньше цены спецпредложенияЗащита от продажи ниже цены спецпредложения
Выбор MIN цены реализацииБерётся минимальная из: дилерская категория, спецпредложение
Не учитывать цену спецпредложенияСпецпредложение игнорируется, работает только дилерская категория

Контроль минимального % наценки при продаже

Если для клиента установлен параметр «Контролировать MIN процент наценки» — система не позволяет продать товар с наценкой ниже заданной в карточке товара. Работает как в BackOffice, так и на СПРУТ POS.

Скидка по позиции на СПРУТ POS

Кассир может изменить уровень цены конкретной позиции (переключить дилерскую категорию). Ограничивается правами пользователя. Все такие изменения фиксируются в журнале.

Раздел 4

Акционное ценообразование

МеханизмОписание
Клиенты по умолчанию по времениВ определённые часы и дни торговая площадка работает как будто все покупают «как клиент Х» — со скидкой этого клиента. Утренние/вечерние акции.
Клиенты по сумме заказаПри достижении суммы чека порогового значения — автоматически применяются скидки «клиента от суммы».
Цены спецпредложения по времениКатегория спецпредложения действует только в указанные дни и часы.
Дисконтные карточки по времениКарточка лояльности действует только в определённые дни и часы.
⚠️
Если у покупателя с дисконтной карточкой скидка по карточке больше чем скидка клиента по времени — применяется большая скидка. Клиент в заказе не заменяется — сумма покупки по-прежнему зачисляется на карточку.

Округление цен при скидках

При применении скидки цена округляется до заданной кратности (барьер / кратность до барьера / кратность после барьера). Работает только при схеме НДС «Вложенный». Применяется только если к товару применена скидка — без скидки округление не выполняется.

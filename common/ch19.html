Инвентаризация

Полная инвентаризация, пересчёт товаров, акты пересортицы, корректировочные накладные.

Раздел 1

Два формата инвентаризации

ФорматОписаниеКогда использовать
Полная инвентаризацияС ведением книг и листов, большим числом участников. Склад замораживается.Плановая ежеквартальная/годовая проверка
Пересчёт товаровУпрощённый формат. Не требует полного замораживания.Текущий контроль по отдельным группам
📱
Оба формата поддерживают работу с ТСД (терминалами сбора данных) — сотрудник сканирует штрих-коды на складе, данные автоматически передаются в систему. Подробно — Глава 27.
Раздел 2

Полная инвентаризация — пошагово

1
Модуль «Инвентаризация» → «Создать новую» → указать склад и дату начала.
2
Система получает остатки по складу на дату начала — это эталонные данные «по системе».
3
Организовать ввод результатов фактического пересчёта: вручную в таблицу или через ТСД.
4
Система сравнивает «по системе» с «фактически» → формирует список излишков и недостач.
5
Выписка корректировочных накладных:
• По излишкам → приходная накладная
• По недостаче → документ-недостача (списание)
• По проблемным → специальная приходная накладная
6
«Обновить остатки по складу» → инвентаризация завершена.
⚠️
Во время инвентаризации — не проводите приходы и продажи по данному складу. Иначе остатки изменятся и результат пересчёта будет некорректным.

Раздел 3

Пересчёт товаров (быстрый формат)

1
Журнал «Пересчёт товаров» → «Создать» → указать группу товаров или склад.
2
Система получает остатки → сотрудники вводят фактические данные (вручную или через ТСД).
3
Статус «Сверка» → руководитель проверяет расхождения.
4
«Подтвердить результаты» → статус «Результаты подтверждены».
5
«Завершить» → автоматически генерируются Акты пересортицы с корректирующими накладными.

Основные отличия пересчёта от инвентаризации

Не требует полного замораживания склада — можно продолжать работу.
Проводится по отдельным группам товаров, а не по всему складу.
Автоматически формирует акты пересортицы при завершении.
Раздел 4

Акты пересортицы

Акт пересортицы — официальный документ, описывающий замену одного товара другим по результатам пересчёта. Группировка товаров в актах производится по альтернативным иерархиям.

Журнал «Актов пересортицы» в модуле «Инвентаризация».
По каждому акту автоматически генерируются корректирующие накладные: списание одного товара + приход другого.
Альтернативная иерархия для группировки задаётся в Свойствах ПС «Спрут» → «Акты пересортицы».

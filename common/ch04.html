Глава 4 · Справочное руководство

Ведение Каталога товаров

Структура карточки товара, единицы измерения, составные товары, иерархия групп, ценообразование, печать ценников. Версия 3.0.

ТовароведОператорКатегорийный менеджерv3.0
Введение

Каталог товаров — основа системы, вокруг которой строится работа всех остальных частей. Ведётся единый каталог для всей группы компаний независимо от архитектуры. Редактирование осуществляется в модуле «Каталог товаров».

Раздел 1

Единицы измерения товара

Каждый товар должен иметь минимум одну — базовую — единицу измерения. Дополнительные единицы связаны с базовой коэффициентом пересчёта.

Рекомендуемые настройки единиц измерения

Тип товараБазовая ед.Ед. по умолчаниюОсн. фасовка
Весовой товарГрамм (г)Килограмм (кг)кг
Жидкий товарМиллилитр (мл)Литр (л)л
Штучный без фасовкиштштшт
Штучный с фасовкойштштЯщик 20 шт
Продажи запрещены — блокирует продажу в данной ед. измерения. Для весовых: продажи в граммах запрещены, в кг — разрешены.
Производный товар — к дополнительной ед. измерения можно привязать отдельный товар-производный (ящик = отдельная карточка товара).
Печатать отдельный ценник — если нужен ценник на нестандартную ед. измерения (например, на ящик отдельно от штучного).
⚠️
Для дробных единиц измерения (кг, л) обязательно установите признак дробности в справочнике «Единицы измерения». Иначе система будет округлять количество до целых, что приводит к ошибкам в весовых товарах.

Последовательность ввода составного товара: сначала создать все компоненты как обычные товары → затем создать составной товар → добавить позиции состава → для каждой позиции можно указать альтернативы (взаимозаменяемые компоненты).

Базовые и производные товары

Используется для товаров в разных формах упаковки: базовый — штучный товар (продаётся в магазине), производный — фасованный (ящик, упаковка для оптовой базы).

ℹ️
При перемещении товаров с оптовой базы в магазин производные товары автоматически конвертируются в базовые. Например: «Молоко 1л ящик 12шт» → 12 × «Молоко 1л».

Связи между товарами

Альтернативы — товары-заменители. Используются в производстве (можно использовать любой из альтернатив) или для аналитики схожих позиций. Задаётся коэффициент пересчёта.
Родственники по штрих-коду — несколько разных товаров с одним штрих-кодом производителя. Каждому присваивается внутренний штрих-код, оригинальный товар помечается «Продажи запрещены».
Уценённые товары — связь между качественным и просроченным/повреждённым товаром. Позволяет уценить часть остатков, не трогая основную карточку.
Сопутствующие товары — для рекомендаций при продаже (интернет-магазин).
Раздел 2

Иерархия групп товаров

Глубина иерархии не ограничена. Рекомендуется не более 4–5 уровней для удобства работы.
Создание группы: выбрать родительскую → «Добавить группу» → задать название.
К группе привязываются Свойства — дополнительные характеристики товаров этой группы (цвет, размер, бренд и т.д.).
Альтернативные иерархии — дополнительные группировки товаров для аналитики, не совпадающие с основной структурой каталога (по сезону, производителю, целевой аудитории).
Раздел 3

Ввод и редактирование товара

1
Выберите нужную группу в иерархии слева → кнопка «Добавить».
2
Заполните обязательные поля: Название, Производитель, Базовая ед. измерения, Операция по НДС.
3
Введите штрих-код или нажмите «Генерировать» для внутреннего.
4
На вкладке «Единицы измерения» добавьте дополнительные ед. с коэффициентами пересчёта.
5
На вкладке «Цены» задайте параметры ценообразования для вашей подгруппы компаний.
6
Сохраните. Артикул генерируется автоматически.
⚠️
Перед вводом нового товара — ищите существующий по штрих-коду и названию. Дублирующие карточки — основная причина пересортицы в системе.

Быстрый поиск в каталоге

По артикулу — введите только код товара (без кода группы).
По штрих-коду — отсканируйте или введите вручную.
По названию — введите часть названия.
Остатки на складе — видны прямо в каталоге при выборе торговой площадки.
История продажных цен — по каждому товару доступна история изменений цен реализации.
Раздел 4

Ввод цен и наценок по дилерским категориям

В карточке товара на вкладке «Цены» задаются параметры ценообразования для каждой подгруппы компаний:

Дилерская категория — базовый уровень цены (Розница, Опт, VIP и т.д.).
Наценка % — процент от приходной цены для расчёта цены реализации.
Мин. / Макс. % наценки — ограничители для автоматической расценки оператором.
Признак фиксирования цены — фиксированная цена не пересчитывается при новом приходе автоматически.
Индикатив — государственная индикативная цена (верхний предел).
Цена спецпредложения — для клиентов определённой категории при достижении объёма.

Подробно вся система ценообразования описана в Главе 10.

Раздел 5

Печать ценников из каталога

1
Выберите товары в каталоге (один или несколько через Ctrl+клик).
2
Кнопка «Печать ценников» → откроется диалог печати ценников (модуль Глава 21).
3
Выберите формат ценника → количество копий → «Печать».
ℹ️
Ценники удобнее печатать прямо из журнала Приходных накладных — сразу при оприходовании товара. Это экономит время при приёмке.
